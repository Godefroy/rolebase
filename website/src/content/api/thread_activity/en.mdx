## Fields

| Field | Type | Description |
| --- | --- | --- |
| `id` | `uuid` | Unique identifier for the activity (auto-generated) |
| `threadId` | `uuid` | Reference to the parent thread |
| `userId` | `uuid` | Reference to the user who created the activity |
| `memberId` | `uuid` | Reference to the member who created the activity |
| `type` | `Thread_Activity_Type_Enum` | Type of activity: Message, Poll, Proposal, ProposalEvent, Thread, Meeting, Task, Decision, or MeetingNote |
| `data` | `JSON` | Activity content (format depends on type) |
| `createdAt` | `timestamptz` | When the activity was created (defaults to current timestamp) |
| `archivedAt` | `timestamptz` | When the activity was archived; null if active |
| `refThreadId` | `uuid` | Optional reference to another thread |
| `refMeetingId` | `uuid` | Optional reference to a meeting |
| `refTaskId` | `uuid` | Optional reference to a task |
| `refDecisionId` | `uuid` | Optional reference to a decision |

## Activity Types

The `type` field accepts the following enum values:

- **Message** ‚Äî Text message or rich content
- **Poll** ‚Äî Poll for gathering opinions
- **Proposal** ‚Äî Governance proposal with prepared org chart changes
- **ProposalEvent** ‚Äî Proposal event (resolution, cancellation, vote reminder)
- **Thread** ‚Äî Reference to another thread
- **Meeting** ‚Äî Reference to a meeting
- **Task** ‚Äî Reference to a task
- **Decision** ‚Äî Reference to a decision
- **MeetingNote** ‚Äî Note created during a meeting

## Relationships

### Object Relationships

- `thread` ‚Äî The thread this activity belongs to
- `user` ‚Äî The user who created the activity
- `refThread` ‚Äî Optional referenced thread
- `refMeeting` ‚Äî Optional referenced meeting
- `refTask` ‚Äî Optional referenced task
- `refDecision` ‚Äî Optional referenced decision

### Array Relationships

- `reactions` ‚Äî Reactions to this activity (see [thread_activity_reaction](/en/api/thread_activity_reaction))

## Query Examples

### List Thread Activities

```graphql
query GetThreadActivities($threadId: uuid!) {
  thread_activity(
    where: { threadId: { _eq: $threadId } }
    order_by: { createdAt: asc }
  ) {
    id
    threadId
    userId
    createdAt
    type
    data
    reactions {
      id
      shortcode
      userId
    }
    refThread {
      id
      title
    }
    refMeeting {
      id
      title
    }
    refTask {
      id
      title
    }
    refDecision {
      id
      title
    }
  }
}
```

## Permissions

Thread activity access is controlled through Hasura permissions and row-level security:

- **View** ‚Äî Users can view activities in threads they have access to
- **Create** ‚Äî Users can create new activities in threads they have access to
- **Update** ‚Äî Users can update their own activities. MeetingNote activities can also be updated by users with access to the thread
- **Delete** ‚Äî Users can delete their own activities. Organization admins and owners can delete any activity

<Callout type="info">
  The `data` field structure varies based on the activity type. Activities are
  ordered chronologically within a thread and support reactions with
  aggregation. The reference fields allow activities to link to other entities
  for context.
</Callout>
