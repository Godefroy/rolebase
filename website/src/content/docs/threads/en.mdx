## Overview

Discussions are Rolebase's tool for **asynchronous collaboration**. They provide a structured space for conversations that do not need to happen in real time, such as governance proposals, project updates, feedback requests, and more.

Each discussion belongs to a role and progresses through defined statuses.

## Creating a Discussion

1. Navigate to a role's discussions section.
2. Click **New Discussion**.
3. Enter a title and opening message.
4. The discussion is created in **Preparation** status by default.

## Discussion Statuses

Discussions move through four statuses:

| Status          | Description                                                                     |
| --------------- | ------------------------------------------------------------------------------- |
| **Preparation** | The discussion is being drafted. Only the creator can see it until it is activated. |
| **Active**      | The discussion is open for participation. All role members can take part.       |
| **Blocked**     | The discussion is paused, waiting for external input or a decision.             |
| **Closed**      | The discussion is concluded and archived for reference.                        |

## Activities and Messages

Discussion activities are the individual messages within a discussion. Members can:

- Post text messages with rich formatting.
- Reply to specific messages.
- Add **reactions** to messages (emoji reactions for quick feedback).

Replies are nested under their parent message, keeping related messages grouped together.

## Polls

Discussions support **polls** for structured decision-making. When you need input from the group, create a poll within a discussion.

### Poll Options

| Option              | Description                                                                                   |
| ------------------- | --------------------------------------------------------------------------------------------- |
| **Multiple Choice** | Participants select one or more options from a predefined list                                |
| **Points**          | Participants distribute a set number of points across options to indicate preference strength |
| **Anonymous**       | Votes are hidden. Results are shown without revealing who voted for what                      |
| **Randomize**       | The order of options is randomized for each voter to reduce order bias                        |

<Callout type="tip" title="Consent-Based Decisions">
  Polls are a great tool for consent-based decision-making in Sociocracy. Create
  a poll with options like "Consent", "Object", and "Stand Aside" to formalize
  rounds.
</Callout>

### Creating a Poll

1. Open a discussion.
2. Click the **Poll** button in the message composer.
3. Add a question and define the options.
4. Configure settings (multiple choice, points, anonymous, randomize).
5. Publish the poll.

Participants see the poll in the discussion and can cast their votes. Results update in real time.

## Discussion Privacy

Discussions can be marked as **private**. Private discussions are only visible to the discussion creator and explicitly added participants. They do not appear in the role's discussion list for other members.

## Pinning a Discussion

You can **pin** an important discussion to keep it at the top of the list for everyone. Use the pin icon in the top right of the discussion to pin or unpin it. Pinned discussions appear first in every discussion list, above unread ones, so the conversations that matter stay easy to find.

## Extra Members

By default, a discussion is visible to all members of its role. You can add **extra members** from other roles to participate in a discussion without giving them full role membership.

This is useful for:

- Cross-role collaboration on specific subjects.
- Getting input from subject-matter experts in other parts of the organization.
- Involving stakeholders who do not need ongoing role membership.

<Callout type="info" title="Notifications">
  Extra members receive notifications for the discussion just like regular role
  members.
</Callout>

## Discussions in Meetings

Active discussions are reviewed during the **Discussions** step of role meetings. This ensures that asynchronous discussions are regularly surfaced in synchronous settings, keeping conversations from stalling.
